This implementation guarantees admin user persistence through a three-layer
safety architecture:

(1) The AdminSeeder uses double-guard logic that checks both table existence
    and emptiness before creating the admin, preventing errors during migrations
    and ensuring idempotency.

(2) The Docker entrypoint script automatically executes migrations and seeders
    on every container startup, eliminating manual intervention.

(3) The seeder is called first in DatabaseSeeder, ensuring the admin role and
    user exist before any other data is seeded.

This approach is production-safe because it leverages Laravel's native seeding
mechanism, adds zero runtime overhead to application requests, provides clear
audit trails through logging, and survives all database lifecycle events while
never touching existing user data.
